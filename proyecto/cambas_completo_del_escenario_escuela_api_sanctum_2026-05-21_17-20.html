Lección de Arquitectura: Flujo de Datos Relacional y CRUD Seguro

Transacciones Atómicas (DB::transaction)

Tanto en `store()`, `update()` como en `destroy()`, el controlador hace uso de DB::transaction. Esto salvaguarda la consistencia relacional. Si ocurre un fallo en mitad de un borrado lógico o físico compleja, el motor de base de datos realiza un Rollback automático, impidiendo que existan estados intermedios inconsistentes.

Prevención Activa del Error de Consulta N+1

Al llamar a Student::with('classroom')->paginate(15), Laravel realiza Eager Loading (carga ansiosa), compilando de manera inmediata las aulas que corresponden a los estudiantes listados. En lugar de ejecutar 1 consulta para estudiantes + 15 consultas individuales para cada aula (un total de 16 consultas SQL), ejecuta únicamente 2 consultas, optimizando el rendimiento de la red y de la base de datos de manera exponencial.

Seguridad con Laravel Sanctum (Bearer API Tokens)

Laravel Sanctum proporciona un sistema de autenticación de pluma para SPAs, aplicaciones móviles y APIs simples basadas en tokens. Al emitir un plainTextToken mediante createToken(), el cliente puede guardarlo y adjuntarlo en cada petición subsiguiente mediante la cabecera estándar de autorización Bearer, la cual es interceptada de forma transparente por el middleware de Sanctum para resolver el usuario autenticado.
